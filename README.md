# 장바구니 + 쿠폰 과제 요구사항

## 목표
상품 선택 화면에서 상품을 장바구니에 담고,  
장바구니 화면에서 담긴 상품/수량/결제 금액을 확인한 뒤 쿠폰을 선택해 최종 결제 금액을 계산한다.

---

## 화면 구성

### Screen 1) 상품 목록 화면
- 더미 상품 리스트를 화면에 출력한다.
- 각 상품 항목에 다음 정보를 표시한다.
  - 상품명, 가격, **재고(stock)**
- 각 상품 항목에서 장바구니 담기 동작을 제공한다.
  - 예: `담기` 버튼, `+` 버튼 등 (형식 자유)

### Screen 2) 장바구니 화면
- 장바구니에 담긴 상품 목록을 출력한다.
- 각 장바구니 아이템에서 다음 정보를 표시한다.
  - 상품명, 단가, 수량, 소계(단가 * 수량)
- 수량 변경 기능을 제공한다.
  - `+` / `-` 버튼으로 수량 증가/감소
  - 수량이 0이 되면 해당 아이템 삭제 처리(또는 삭제 버튼 제공)
- 결제 금액 영역을 표시한다.
  - **총 상품 금액**(쿠폰 적용 전)
  - **할인 금액**
  - **최종 결제 금액**(쿠폰 적용 후)

---

## 쿠폰 기능 요구사항
- 더미 쿠폰 리스트를 화면 하단(또는 적절한 위치)에 표시한다.
- 사용자는 쿠폰을 1개 선택할 수 있다. (선택 없음 포함)
- 쿠폰은 **퍼센트 할인(PERCENT)** 만 구현한다.
- 쿠폰 적용 시:
  - 할인 금액 = `총 상품 금액 * (discountValue / 100)`
- 최종 결제 금액 = `총 상품 금액 - 할인 금액`

---

## 선택 과제(추가 구현)
- **최소 주문 금액(minOrderPrice) 조건 처리**
  - 총 상품 금액이 `minOrderPrice` 미만이면 해당 쿠폰은 **사용 불가** 처리
  - 사용 불가 표현 방식은 자유 (비활성화, 안내 문구, 선택 시 경고 등)
- **최대 할인 금액(maxDiscountPrice) 상한 처리**
  - 계산된 할인 금액이 `maxDiscountPrice`를 초과하면 할인 금액을 `maxDiscountPrice`로 제한한다.
  - 예: 20% 할인으로 18,000원이 할인되어야 하지만 `maxDiscountPrice = 15,000`이면 할인 금액은 15,000원
- **재고(stock) 조건 처리**
  - 상품에 `stock` 값이 존재한다고 가정한다.
  - **상품 목록 화면에서 재고(stock)를 함께 표시한다.**
  - 장바구니 담기/수량 증가 시, 해당 상품의 장바구니 수량이 `stock`을 초과할 수 없다.
  - 초과 시 처리 방식은 자유 (동작 무시, 안내 문구/토스트 표시 등)

## 데이터
- 제공된 더미 JSON을 사용한다.
  - `products`: 상품 리스트
  - `coupons`: 쿠폰 리스트
- 네트워크/API 연동은 하지 않는다. (로컬 더미 데이터만)
